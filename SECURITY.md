# Security Policy

We take security issues seriously. If you find a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

## Supported Versions

Only the latest release version of AlgoVisionCPP receives security updates. If a vulnerability is found, we will patch the main branch and release a minor/patch version as soon as possible.

| Version | Supported |
| ------- | --------- |
| v1.0.x  | Yes       |
| < v1.0  | No        |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues. 

Instead, report vulnerabilities privately by emailing the maintainer at `sec-report@algovisioncpp.org` (or directly via Tanmay Tyagi's security channels).

When reporting, please include:
- A description of the vulnerability and its potential impact.
- Step-by-step instructions to reproduce the issue.
- Details of your development environment (OS, compiler, SFML version).

We will acknowledge receipt of your report within 48 hours and work with you to analyze and patch the issue promptly.
